On the cloud, you are constantly moving instances around, starting new ones or terminating unused servers. If you are using Amazon EC2, DNS management can be an operational nightmare. For every change you make to your infrastructure, new IP addresses are generated that you have to update manually.- 日本語 - クラウド上で常にインスタンスが起動したり、停止したり、移行してます。もしアマゾンEC2をご使用になられてたら、DNS管理の運用は悪夢に感じるかもしれません。インフラに新たな変更を食わせるたびに新たなIPアドレスが発生し、これらを手動で更新しなければなりません。These difficulties were partly solved by Amazon with the introduction of Elastic IPs. However, most of the time, your application, your load balancer or your database servers communicate using private IP addresses. So far, Amazon EC2 does not provide the tools to automate the management of your DNS records.- 日本語 - Elastic IPの導入でこの課題が一部解消されましたが、アプリ・LB・データベースサーバはプライベートIPアドレスで話し合います。アマゾンEC2はDNSレコード管理を自動化するツールを提供してません。Amazon EC2 DNS management made easy- 日本語 - アマゾンEC2のDNS管理を簡易化へFortunately, Scalr can manage your DNS zones for you, should you wish to have it do so. It is as easy as logging in to your registrar and point your domain (or subdomain) to Scalr’s name servers. DNS zones are automatically updated by Scalr to reflect the Amazon EC2 instances you are running at any given time.- 日本語 - ScalrはDNSゾーンを管理してくれます。レジストリにログインし、Scalrのネームサーバにドメイン（サブドメインも可）を登録すれば終了です。Scalrは稼働中のアマゾンEC2インスタンスを反映すべき、自動でDNSゾーンを更新します。- When a new instance is brought up, its IP address is added to the DNS zone as an A record.- When an instance is terminated, its IP address is removed from the DNS zone.- If an instance has an ElasticIP, that IP is used for the record.- When an instance with an ElasticIP dies, the ElasticIP is used for the record of the replacement instance (to which the ElasticIP is attached).- 日本語 - - 新たなインスタンスが起動されたら、インスタンスのIPアドレスがAレコードとしてDNSゾーンに追加されます。- インスタンスが停止されたら、IPアドレスがDNSゾーンから削除されます。- インスタンスがElasticIPを持ってたら、そのIPアドレスがレコードとして使われます。- ElasticIP持ちのインスタンスが死んだ場合、同ElasticIPが代用インスタンスに付与されます。It’s THIS easy!Don’t just take our word for it:- 日本語 - これだけ簡単なのです！Scalrを使ってみましょう。
