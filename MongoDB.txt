You love MongoDB for its scalability and developer agility. Scalr is happy to offer public support for MongoDB and will, as always, make your life easier: enjoy self-managed, easily scaled, configurable Mongo.
MongoDBがスケーラビリティに優れていて、俊敏な開発が出来るので気に入っているでしょう。Scalrは公式にMongoDBのサポートをし、作業がより簡易的にできるよう心がけてます。自己管理可能な、簡単にスケールできる、構成できるMongoをお楽しみください。

Scaling
スケーリング

Use Scalr to add new shards or replicas set for extra read and write capacity.
Scalr使用で読み書き可能な新パーツやレプリカを追加できます。

Deploy your cluster across Azs
アベイラビリティゾーンにクラスターを配置

Scalr automatically provisions nodes across different availability zones. Set “placement & type” to “Distribute equally” in the farm’s settings to ensure that your instance are spread across multiple AZs.
Scalrは異なったアベイラビリティゾーンへ自動的にノードを配置する事が出来ます。

異なったアベイラビリティゾーンにインタンスが分散されるようにファームの”placement & type”を “Distribute equally”に設定してください。"

Persistence with EBS volumes
EBSボリュームによる持続性

 Scalr stores data on EBS volumes, so if some instances go down, the volumes are automatically mounted on replacement ones.
"ScalrはEBSボリュームにデータを置いているので、インスタンスがダウンしても自動的に代わりのインスタンスにマウントします。"

Within Scalr you can change all your MongoDB database settings
ScalrではMongoDBのデータベースすべての設定を変更する事が出来ます。

Like all database roles, the MongoDB options menu contains a MongoDB status page.
"他の全てのデータベース役割設定と同様に、ScalrのMongoDB options menuには、
MongoDBのステータスページがあります。"

The MongoDB status page allows you to add new shards and replicas.
MongoDBのステータスページでは、新しいパーツやレプリカを追加する事が出来ます。
